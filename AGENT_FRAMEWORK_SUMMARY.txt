                           DESIGN HIGHLIGHTS
================================================================================

1. MANDATORY VERIFICATION
   - Every agent's verify() method must validate its output
   - Catches command failures, partial output, timeout data issues
   - Production-quality validation prevents bad results

2. APPROVAL WORKFLOW
   - Automatic detection of risky operations
   - Tasks await operator approval before execution
   - Full audit trail with approved_at and approved_by

3. ASYNC/AWAIT THROUGHOUT
   - All agents use async execute()
   - Timeout enforcement via asyncio.wait_for()
   - Concurrent execution with semaphore control

4. COMPREHENSIVE LOGGING
   - Task start/complete with duration
   - Verification results logged
   - Failed operations with full error context
   - Rich console formatting for development

5. DATABASE INTEGRATION
   - Execution history persisted to TaskRecord
   - Queryable by task_type, status, date
   - Cost tracking with UsageLog model

6. SANDBOXING

   - File: Allowed directory whitelist
   - Code: Subprocess isolation with temp files
   - Web: URL validation and rate limits

7. TASK ROUTING
   - Registry-based agent discovery
   - Dynamic agent lookup by name
   - Automatic capability listing
   - Category-based filtering

8. ERROR HANDLING
   - Validation before execution

================================================================================
                            HOW TO USE
================================================================================

1. BASIC EXECUTION:
   curl -X POST http://localhost:8000/api/v1/agents/execute \
     -H "Content-Type: application/json" \
     -d '{
       "agent_name": "shell_agent",
       "instruction": "List files",
       "params": {"command": "ls -la"},
       "timeout_seconds": 30
     }'

2. GET CAPABILITIES:
   curl http://localhost:8000/api/v1/agents/capabilities

3. APPROVE A TASK:
   curl -X POST http://localhost:8000/api/v1/agents/approve/TASK_ID \
     -H "Content-Type: application/json" \
     -d '{"approved_by": "admin", "notes": "Approved"}'

4. GET HISTORY:
   curl "http://localhost:8000/api/v1/agents/history?limit=50"

5. BATCH EXECUTION:
   curl -X POST http://localhost:8000/api/v1/agents/batch \
     -H "Content-Type: application/json" \
     -d '[
       {"agent_name": "shell_agent", "params": {"command": "ls"}},
       {"agent_name": "shell_agent", "params": {"command": "pwd"}}
     ]'
