3. CORE DATA FLOWS
================================================================================

FLOW 1: Chat Request Processing
─────────────────────────────────────────────────────────────────────────

ChatRequest → orchestrator.py:process_chat()
  ├─ Normalize spelling (_normalize_for_matching)
  ├─ Detect data queries
  │   ├─ Stock prices? → yfinance.fetch_live_stock_data()
  │   ├─ News? → brave_search.fetch_live_search_data()
  │   └─ Other → pass to LLM
  ├─ Retrieve memories (memory.py:recall_memories) via Chroma
  ├─ Analyze images (vision_service.py) if included
  ├─ Route to LLM (llm_router.py:chat_completion)
  │   ├─ Model selection based on tier (reasoning/fast/auto)
  │   └─ Auto-route based on complexity
  ├─ Execute tools if requested (tool_calling.py)
  ├─ Validate response (fact_checker.py)
  └─ Generate recommendations (recommendations.py)

Response → ChatResponse (with metadata, confidence, images, warnings)


FLOW 2: Agent Execution
─────────────────────────────────────────────────────────────────────────

AgentTaskRequest → agents/models.py:AgentTask
  → executor.execute(task)
    ├─ Check approval requirements
    ├─ Validate task (agent.validate)
    ├─ Execute (agent.execute)
    ├─ Record to history
    └─ Return AgentResult

Streaming variant emits progress events via SSE:
  event: progress → agent is processing
  event: bash → command executed
  event: chain → agent invoked another agent
  event: result → final outcome


FLOW 3: Voice Session
─────────────────────────────────────────────────────────────────────────

POST /voice/live/start
  → VoiceSessionManager.create_session()
  → Returns session_id + config

POST /voice/live/turn (repeat)
  ├─ record_user_turn()
  ├─ build_voice_system_overlay()
  ├─ get_conversation_context()
  └─ Return context for /chat/stream with voice overlay

POST /voice/live/interrupt → session.interrupt()
POST /voice/live/end → mgr.end_session() + stats


FLOW 4: Scan & Briefing Generation
─────────────────────────────────────────────────────────────────────────

Automatic (cron) or Manual:
  POST /scanner/scan-now
    → orchestrator.run_full_scan()
      ├─ Iterate through configured sources
      │   ├─ NewsAPI
      │   ├─ Twitter/X API
      │   ├─ Reddit
      │   └─ Custom sources
      ├─ Filter by keywords + relevance threshold
      └─ Store results

GET /scanner/briefing
  → orchestrator.get_briefing()
    ├─ Load recent scan results
    ├─ Compile into markdown
    ├─ Use LLM to generate summary + insights
    └─ Cache for repeated requests


FLOW 5: Memory (Vector Embeddings)
─────────────────────────────────────────────────────────────────────────

store_memory(content, metadata)
  → Chroma.add()
    ├─ Generate embedding (OpenAI or Sentence Transformers)
    └─ Store vector + metadata

recall_memories(query)
  → Chroma.query()
    ├─ Generate embedding for query
    ├─ Semantic similarity search
    └─ Return top-k results (default k=5)

4. ORCHESTRATOR (THE BRAIN)
================================================================================

File: /sessions/admiring-funny-gauss/mnt/cipher-app/app/services/orchestrator.py

Purpose: Central hub that coordinates all chat processing

Key Functions:

process_chat(request, db) → ChatResponse
  ├─ 1. Data Detection
  │     └─ Patterns: stocks, news, search, time-sensitive info
  │     └─ If matched: invoke real agents BEFORE LLM
  │
  ├─ 2. Spelling Correction (deCIPHER)
  │     └─ Normalize typos: "analize" → "analyze"
  │     └─ Ticker shortcuts: "tsla" → "TSLA"
  │
  ├─ 3. Memory Integration
  │     └─ Recall relevant context from Chroma
  │     └─ Inject into system prompt
  │
  ├─ 4. Image Analysis (if images in request)
  │     └─ Call vision_service.py → Claude Vision API
  │     └─ Inject descriptions into prompt
  │
  ├─ 5. LLM Routing
  │     └─ Call llm_router.py:chat_completion
  │     └─ Select model (reasoning/fast/auto)
  │     └─ Handle context window, tool calling
  │
  ├─ 6. Tool Execution
  │     └─ If LLM requests tools (search, web, etc.)
  │     └─ Execute via tool_calling.py
  │     └─ Feed results back to LLM
  │
  ├─ 7. Response Validation
  │     └─ fact_checker.py: Check unverified claims
  │     └─ Generate confidence_score (0-1)
  │     └─ Add validation_warnings
  │
  ├─ 8. Recommendation Generation
  │     └─ recommendations.py: Suggest agents
  │     └─ "Let Crawler scrape that?"
  │
  └─ 9. Response Assembly
        └─ Save to DB (ConversationRecord, MessageRecord)
        └─ Return ChatResponse with all metadata

Key Patterns Detected:

_STOCK_PATTERN
  → Matches: "price", "quote", "$TSLA", ticker symbols
  → Action: Fetch live price via yfinance
  → Result: Inject real prices into prompt

_NEWS_PATTERN
  → Matches: "breaking news", "what's happening", "latest"
  → Action: Brave Search API
  → Result: Top 5 recent articles

_SEARCH_PATTERN
  → Matches: "search for", "look up", "how much/many"
  → Action: Web search or agent invocation

7. STREAMING & REAL-TIME
================================================================================

Server-Sent Events (SSE) used for:

1. /api/v1/chat/stream
   Events: token, image, metadata
   Pattern: Chunks text in 3-char pieces
   Keepalive: Every 5 seconds if processing

2. /api/v1/agents/execute/stream
   Events: progress, bash, chain, result
   Pattern: Real-time task updates
   Keepalive: Heartbeat if idle

3. /api/v1/voice/synthesize/stream
   Pattern: Chunked audio transfer

Client Implementation (JavaScript):

const eventSource = new EventSource(url);
eventSource.addEventListener('token', e => {
  const data = JSON.parse(e.data);
  // Append to UI
});
eventSource.addEventListener('metadata', e => {
  // Final response
  eventSource.close();
});
